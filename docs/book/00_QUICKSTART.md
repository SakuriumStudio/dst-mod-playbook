# 00 クイックスタート（結論だけで勝つ）

「仕組みは後でいい、まず表示させたい」用の章。

## 0-1. まず前提
DSTのUI画像は多くのケースで **.tex + atlas .xml** が前提。  
PNGだけ置いても、ゲームが探すのは **atlas（xml）と region（Element name / tex名）** なので出ないことがある。

## 0-2. 5秒チェック（ここで8割決まる）
1) 目的のフォルダに **.xml と .tex がある？**（PNGしかないならまずそこ）  
2) client_log.txt で **探してる atlas/region の文字列** を拾う（出てたらそれが“仕様”）  
3) bigportrait系は ***_none / *_none_oval** を探すことが多い  
4) スキン有効だと **prefab名 → skin id** 側に参照キーが切り替わることがある  
5) atlasの <Element name="..."> が **探されてる region 名** と一致してる？

## 0-3. 最短の勝ち筋
- いちばん安全：**ゲームが探す region 名に合わせて PNG名（＝生成されるElement name）を合わせる**  
- 既に生成物があるなら：xmlの <Element name> を要求名に合わせて直す（運用で吸収）
